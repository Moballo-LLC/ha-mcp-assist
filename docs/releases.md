# Releases

HA MCP Assist uses GitHub releases for release notes and HACS package delivery.
The repository does not maintain a separate `CHANGELOG.md`.

## Release Notes

Release notes are generated by GitHub from merged PRs using:

```text
.github/release.yml
```

To keep release notes useful:

- Use clear PR titles.
- Keep PRs small and atomic.
- Apply labels consistently when release-note grouping matters.
- Put user-facing context in the PR body when a title alone is not enough.

## Version Source

The integration version lives in:

```text
custom_components/mcp_assist/manifest.json
```

The release workflow requires the Git tag version to match the manifest version.

Example:

- manifest version: `1.0.0`
- tag: `v1.0.0`

## Release Workflow

The release workflow lives at:

```text
.github/workflows/release.yml
```

On `v*` tags, it:

1. Checks that the tag starts with `v`.
2. Checks that the tag version matches `manifest.json`.
3. Builds `dist/mcp_assist.zip` from `custom_components/mcp_assist`.
4. Publishes the GitHub release.
5. Attaches the HACS zip artifact.

## How to Publish a Release

1. Make sure the release PRs are merged.
2. Update `custom_components/mcp_assist/manifest.json` with the new version.
3. Merge the version bump.
4. Create and push a matching tag:

```bash
git checkout main
git pull origin main
git tag vX.Y.Z
git push origin vX.Y.Z
```

5. Watch the **Release** workflow.
6. Review the generated GitHub release notes.

## Versioning Guidance

Use judgment based on user impact:

- Patch: bug fixes, docs, CI, internal cleanup, small compatibility fixes.
- Minor: new tools, new providers, new settings, or meaningful behavior
  improvements.
- Major: breaking changes, migration requirements, or compatibility drops.

Runtime compatibility should stay lenient unless there is a proven reason to
raise a minimum version.

## HACS Package Shape

The release artifact is built from:

```text
custom_components/mcp_assist
```

Do not rename the integration folder or manifest domain as part of a release.
HACS and Home Assistant can display this project as **HA MCP Assist** while the
integration still loads from `custom_components/mcp_assist` with domain
`mcp_assist`.

## Pre-Release Checklist

- `custom_components/mcp_assist/manifest.json` version is correct.
- CI is green on `main`.
- HACS and Hassfest validation pass.
- The release notes generated from merged PRs read well.
- Any user-facing docs changed with the feature or behavior change.
- Runtime dependency bounds remain as lenient as practical.
